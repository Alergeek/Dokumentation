Description

Almost everything happens in the MainActivity.

1) We activate our voiceregocnition and checking if we already have a token saved on the Vuzix.

2) We start the scan Activity if we have no token saved right now we get a toast.

3) In the onActivityResult method we got two possibilities: 3.1) The Token wasn't set so we call the PostSession class 3.2) The Token was set so we can call the GetProduct class But always with the scaned ean as param.

4)The GetProduct class make a HTTPRequest to get the edible-status of the scanned Product. We check the Response statuscode If it is equals to 200 we know that the request was succesfull and we received a result. Else if the code was equals 401 or 404 we delete the Token or print a Toast because the scanned Product is not checkable.

5) The PostSession class is working really simular to the GetProcduct class . The only diffrent is, that we now thet the NameValuePair as param for the PostRequest and saving the result as the new Token.
